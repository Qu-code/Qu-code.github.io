Hexo搭建个人博客

发表于 2020-01-01 更新于 2020-01-23 分类于 Hexo博客 阅读次数：

Hexo是一个快速、简洁且高效的博客框架。利用Hexo框架可以快速搭建属于自己的博客，创造一片仅属于自己的网络空间。这篇博客介绍了windows操作系统如何使用Hexo创建博客，并利用Github让你的博客发布到互联网上。

安装Hexo博客

安装Hexo并不需要繁杂的操作和大型软件，只需安装下列应用程序即可。

Node.js + Git

准备工作主要就是安装Node.js和Git。Node.js是一个基于Chrome V8引擎的Javascript运行环境，提供了一个让JavaScript运行在服务端的平台。Git是一个开源的分布式版本系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git是为了帮助管理Linux内核开发而开发的一个开放源码的版本控制软件。

  • Node.js（Node.js 版本需不低于 8.10，建议使用 Node.js 10.0 及以上版本）
  • git

中国大陆用户用上述网址下载安装包可能会十分缓慢，我建议用下列淘宝镜像网址进行下载。

  • Node.js
  • git

安装过程

安装过程十分简单，选定好安装文件夹，一路next下去就可以了。不过在安装时需要注意以下两点。

  1. 安装Node.js的时候建议勾选Add to path，这样就避免了后续需要自己添加环境变量的麻烦。
  2. 安装Git时，注意勾选Git Bash Here

开始搭建博客

所有必备的应用程序安装完成后，就可以开始搭建个人博客了。首先建立一个空文件夹，打开文件夹点击右键选择Git Bash Here打开命令行

在命令行中输入如下指令安装hexo博客
$ npm install -g hexo-cli
经过一段时间的下载和安装，hexo博客就成功安装了。

需要注意的是，中国区的用户使用npm可能速度会很慢，我们可以首先下载淘宝npm镜像
$ npm install -g cnpm --registry=https://registry.npm.taobao.org
安装好之后，便可以使用cnpm安装Hexo。由于本文挑战发起者位于加拿大，可以忽略这里，使用npm指令。
$ cnpm install -g hexo-cli

安装完成后便可以正式开始建立自己的博客，在命令行中输入如下指令：
hexo init
这时你就会发现刚才所创建的空文件夹中多出了一些文件夹和文件。具体每个文件夹以及文件对应什么功能，我们在这里就不作深入讨论了。我可以告诉你的时所有博客的内容都将会包含在这个文件夹中，当你经过一段时间的捣鼓反而造成了不可恢复的错误时，不要惊慌，删除掉这个文件夹重新来过即可。

你可以输入
hexo -v
查看hexo 博客的版本

生成静态页面
hexo g
启动博客
hexo s

这样一个博客就搭建完成了，你可以打开浏览器，输入网址localhost:4000来浏览你的网页。你所创建的初始网页讲如下图所示，并且还会有一篇hexo所写的博文。

部署博客

这样搭建的博客只能在本地电脑上显示，这样显然是不够的。要想让自己的博客被他人浏览还需要将博客部署到远端服务器上。对于家境贫寒的我来说，每个月支付一笔钱来维持服务器或是在家里自己搭建一台服务器主机都是不现实的。不过好在很多平台都提供了免费的部署服务。下面我仅介绍将博客部署到GitHub上的方式。Github是一个面向开源及私有软件项目的托管平台。该平台已经成为世界各地的人互相交流、学习的大型平台，相信任何程序员都会有在这个平台上学习交流的精经历。我们所使用的是Github所提供的GitHub Page服务，它是免费的静态网页托管服务，可以用来托管博客、项目官网等静态网页。这无疑是我们家境贫寒之人的福音。话不多说，我们开始把！

配置SSH Key

要使用git工具首先要配置SSH Key。

登录GitHub创建仓库

首先请注册一个GitHub账号，登陆后创建一个新的仓库用来存放博客内容。

新建时仓库名称为xxx.github.io,xxx必须为自己的用户名，如下图所示。描述里面可以自由发挥，然后直接创建就可以了。

生成SSH Key

配置Guthub用户信息
git config --global user.name "GitHub用户名"
git config --global user.email "GitHub邮箱"
在命令行中执行下面命令，生成ssh key
ssh-keygen -t rsa -C "注册GitHub的邮箱"
然后一路回车下去就可以了，下图中由于我已经生成了，所以我输入了n，没有ssh key的输入y就可以了。这样

生成的ssh key文件在上图红框中。进入改文件夹，将公钥部分配置到刚才所创建的GitHub仓库的ssh key中去。该文件用记事本即可打开。
进入GitHub个人设置，找到SSH and GPG keys这一栏，选择New SSH key,然后将刚才所复制的公钥的内容拷贝进去即可，title可以随意填写。

这样SSH key就配置完成了，可以在命令行中输入下述指令来确认配置成功
ssh git@github.com

安装远程部署插件

在命令行中输入
npm install hexo-deployer-git --save

修改配置文件

打开你为博客所创建的文件夹，文件夹中有一个名称为_config.yml的文件，这个文件里变存放着一些博客的基础信息。你需要修改网址的名称，主题，博主的名字，或是头像，都是在这个文件中进行更改。网上有部分教程是利用命令行对文件进行更改，这里我并不赞同没有Linux基础知识的人用这一方式，因为这一方式确实比较麻烦。我比较推荐下载Visual Studio Code这一编辑器进行操作。这一编辑器不仅可以操作这类文件，后面撰写博文也需要用到markdown编辑器，而这一编辑器也可以胜任这一工作。如果实在不愿意下载新的软件，那用记事本打开进行修改也可以。
打开_config.yml文件，修改deploy信息

这样准备工作就做好了。

部署博客

生成博客
hexo g
部署博客
hexo d
这样博客就部署完成了，博客的地址就是仓库的名字。打开浏览器，输入地址，便可以在网上浏览自己的博客了！需要注意的是，部署博客需要一段时间，可能几秒，也可能几分钟。如果你修改了博客内容并部署成功，然后兴奋地去网页查看发现却没有发现变化，这时不要心急，等几分钟再看看。

更换博客主题

很多为hexo博客制作了精美的主题，用户只需要下载主题，然后更换主题即可。目前用的比较多的主题有next主题和yilia主题。我所使用的就是next主题。下面我以next主题为例，介绍如何给博客更换主题。

下载主题

命令行中输入后等待下载
git clone https://github.com/theme-next/hexo-theme-next themes/next

配置主题

打开_config.yml文件,将theme后面内容修改为next。这里一定要注意空格，千万不要在原有基础上增加空格或减少空格。

然后重新生成一下博客就可以了。
hexo g
输入启动博客，并在本地看看是否修改成功，如果修改成功，那么就可以部署到远端了。
hexo s

撰写博客

放在博客上的博文应该是markdown格式的文件。markdown基本语法非常简单，这里我也不进行介绍了。将写好的博文存放到blog\source_post文件夹中，输入命令生成博客就可以了。
hexo g

后记

这是我的第一篇技术博客。接受了来在TM的challenge，希望你看了这篇博客可以顺利地搭建自己的博客。内容有不足不清楚之处还请多多包含指正。
在19年11月考完二级笔译之后我本觉得自己应该可以顺利通过，于是我就开始安排后面的学习计划。之前一直都想自己维护一个网站，于是我想着从简单的搭建博客开始入手，学习一下网页前端的知识，同时进行基础语言的学习。谁知道正当我研究前端知识时，二级笔译的滑铁卢打乱了我的计划。20年1月15日成绩在网上公布，最终没能通过考试。于是今年还得继续二级笔译的复习。经过分析和查阅了一些经验贴，我认为是我的英译汉存在不足。翻译和阅读理解不一样，译出来的译文并不只是把作者表达的意思写出来就行了。翻译就是写作！英译汉不是对英语的考察，更多的是对汉语的考察。所以后面虽然博客计划被我搁置了，但是我还是会坚持发布自己的博文和大家交流。主题可能多种多样，但是技术贴可能就没有了。所以这是我第一篇技术博客，也将是我近期的最后一篇技术博客。后面我将会围绕多个主题进行分享、写作。本人水平有限，内容看法一定会存在肤浅的地方，这时就希望读者朋友们（主要就你们几个知道我地址的lol）给我提出宝贵意见。
最近新型冠状病毒肆虐，我的家乡武汉是受灾最严重的城市，同时也是引起其他城市遭受灾难的源头。对于家乡的遭遇，我感到十分难受，不过每天都看到医护人员和各界人士的暖心故事也让我为之一振。这让我想到基辛格的《论中国》里的一句话-中国总是被他们最勇敢的人保护得很好。这些平凡的医护人员面对危险毫不退缩，勇于担当，为武汉人民的健康提供了有力保障。一方有难，八方支援。其他省市的医护人员也纷纷写下志愿书，希望能来到武汉贡献自己的力量。最近每天都有更多的人感染病毒，也有更多的人离世。这些消息让今年春节不再那么欢庆。但是我相信中国的医护人员，相信广大的中国人民有信心有能力度过难关。
